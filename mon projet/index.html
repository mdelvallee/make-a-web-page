Lesson 1 : The basics of the web and html

How the web works

The WEB is a bunch of computers that communicate with each other. When a person goes to a web page like Google, their computer sends a HTTP Request to a server.
SERVERS are computers optimized for hosting data and HTML or HyperText Markup Language is the protocol used for communicates between all those elements
The server finds the appropriate HTML document and sends it back to the user's computer where a WEB BROWSER, a local software on everyone's computer, interprets the page and displays it on the user's screen.
This video does a good job of explaining.

What a web page is

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.
HTML documents contain text content which describes "what you see" and markup which describes "how it looks". This video gives a good overview.
HTML documents are made of HTML tags, elements and attributes.
When writing HTML, we tell browsers the type of each element by using HTML tags. This video explains the distinction well. An HTML tag is always contained within angled brackets. Most tags have an opening tag <p> for example and a closing tag, </p>. Some tags (called "void" tags) do not require a closing tag (like the <br> tag).
Attributes are the properties of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write <a href="www.udacity.com">

How coding works

Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote.
There are too many details to keep everything in head but search engine are our friends. Coding is a state of mind, knowing how to speak to a computer, with the appropriate language.

Lesson 3 : Adding CSS Style and HTML Structure

What is CSS and why using it

CSS is code written to control the "style" of HTML elements.
You can also think about CSS as a search and replace tool: you identify a class or a tag of the element you want to find (or match, in CSS terminology), and then what you want to do with it, or what property values to replace with different ones.
The code will be lighter using this, avoiding a lot of repetitions so reducing potential mistakes.
The page will also be more consistent with all elements of the same class with the same look

How to add CSS style to a webpage

Possible but bad idea: Write youre style inline with your HTML

<body>
<div style="background-color: red; color: white">
This div will have a red background and white text.
</div>
...
this div will have a red background

This is a bad idea because it leads to lots of repeated code and lots of added work. So avoid this method as much as possible.


Acceptable for very small projects : Write CSS in the <head> section of your HTML :

<head>
<style>
div {
}
</style>
</head>

Mostly used method : Link your HTML to a separate CSS file by writing the link address in your <head> section of your HTML.

<head>
<link rel="stylesheet" type="text/css" href="main.css">
</head>

The link rel="stylesheet" portion links the css document to the html as long as it is located in the same folder.

How does CSS "control" HTML?

If you want to write CSS that makes all h1 elements have a black background and white text, you would write:

h1 {
background-color : black;
color: white;
}

In this example, the h1 is a selector. It says to the browser "I want you to apply the rules I'm about to tell you to every h1 element."
After the h1 there is a left curly brace { and at the bottom there is a matching right curly brace }. Everything between these curly braces will be interpreted as a "rule" that should be applied to every h1 element.
The line of code that says background-color : black; is a declaration. background-color is a property and black is the value of that property

In the example above, we set the background-color of every h1 to black. If we only want to add style to certain h1s, we can use class selectors.

html :

<div class="stop"></div>
<div class="slow"></div>
<div class="go"></div>

with css :

div {
height : 50px;
width : 50px;
border-radius: 25px;
}
.stop {
background-color: red;
}
.slow {
background-color: yellow;
}
.go {
background-color: green;
}
gives :

We can apply rules to all divs or just certain divs by using the appropriate selector.

Inheritence

Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing the software developers to write less CSS. It enhances faster-loading of web pages by users and enables the clients to save money on bandwidth and development costs.
Descendent elements inherit text-related properties (color, font, ...) but not box-related properties (background, border,...).

Box sizing and Positioning

HTML elements are boxes and each box has 4 components
Because there are so many components to each box, it can often be hard to get the size of a box just right.
There are two techniques you can use to help deal with sizing issues.
Set sizes in terms of percentages rather than pixels.
Set the box-sizing attribute to border-box for every element. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently so we need to add some lines for compatibility :

* {
-webbox-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
